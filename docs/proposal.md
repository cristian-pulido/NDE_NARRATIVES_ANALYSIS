# Structured Coding of NDE Narratives with Human and LLM Annotation

## Objective

This repository supports the analysis of narrative reports of Near-Death Experiences (NDEs) collected in a survey dataset. The goal is to extract structured information from three narrative sections so those annotations can be compared with questionnaire responses and with outputs generated by large language models.

## Narrative Units

Each participant provides three sections:

1. `context`: circumstances surrounding the event
2. `experience`: phenomenological description of the NDE
3. `aftereffects`: perceived consequences following the experience

The current study design assumes complete narratives across all three sections.

## Measures Linked to the Narratives

- `M8` / NDE-C: phenomenological characteristics of the NDE
- `M9` / NDE-MCQ: moral and cognitive changes after the experience

## Structured Outputs

The coding framework extracts:

- Tone for each section: `positive`, `negative`, or `mixed`
- Five M8-related elements from the `experience` section:
  - `m8_out_of_body`
  - `m8_bright_light`
  - `m8_peace`
  - `m8_time_distortion`
  - `m8_presence`
- Five M9-related elements from the `aftereffects` section:
  - `m9_moral_rules`
  - `m9_long_term_thinking`
  - `m9_consider_others`
  - `m9_help_others`
  - `m9_forgiveness`

All extracted elements are binary: `yes` or `no`.

## Why the Repository Uses Three Separate Prompts

The source dataset contains three distinct narrative sections that correspond to different stages of the experience. The repository therefore keeps prompt generation and evaluation section-specific instead of concatenating the narratives into a single block.

This makes it possible to analyze trajectories such as:

- negative context -> positive experience
- positive experience -> positive aftereffects
- mixed trajectories

## Questionnaire Binarization Rules

### M8

- `Medium`, `Intensely`, `Extremely` -> `yes`
- `Not at all`, `Slightly` -> `no`

### M9

- `Increased`, `Strongly increased` -> `yes`
- `Strongly diminished`, `Decreased`, `Not changed` -> `no`

## Evaluation

The repository calculates:

- Accuracy
- Cohen's Kappa
- Macro F1

The intended comparisons are:

- human annotations vs. LLM predictions
- human annotations vs. questionnaire-derived binary labels
- LLM predictions vs. questionnaire-derived binary labels
